🚀 One-Liner Commands (Manual)

Варіант А: Чистий Git (Стандарт)

Найкраще підходить для локальної розробки, але потребує браузера для завершення PR.

  1. Stage: git add .
  2. Commit & Push: git commit -m "Your comment" && git push
  3. Branch names & Switch: git branch --sort=-committerdate (список) → git checkout <name>
  4. PR & Merge: git request-pull (складно) → Краще відкрити GitHub у браузері.

Варіант Б: GitHub CLI (gh) (Професійно)

Це найкращий вибір, оскільки він дозволяє завершити весь цикл (включаючи PR та Merge), не прибираючи рук з клавіатури.

  1. Stage: git add .
  2. Commit & Push: git commit -m "Your comment" && git push
  3. Switch to recent: git checkout $(git branch --sort=-committerdate --format="%(refname:short)" | head -n 2 | tail -n 1) (авто-перехід на останню використану).
  4. PR & Merge: gh pr create --fill --base main && gh pr merge --squash --delete-branch

[!note] TIP: Використовуйте git для коммітів, а gh для всього, що стосується взаємодії з репозиторієм на сервері.
