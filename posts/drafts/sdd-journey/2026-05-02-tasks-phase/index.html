Розбиття Плану на Атомарні Задачі (Tasks Phase)

Після успішного завершення етапів Clarification та Planning, ми перейшли до найважливішої частини підготовки перед написанням коду — створення списку завдань (Tasks Generation). Цього разу ми застосували спеціальний ADHD-friendly підхід, щоб зробити артефакти максимально читабельними та зрозумілими.

Що таке ADHD-friendly список завдань?

Для розробника (особливо при роботі в соло або в парі зі штучним інтелектом) важливо уникати когнітивного перевантаження. Тому наш tasks.md має наступні особливості:

  • 🎯 Чіткі цілі для кожної фази.
  • 🏁 Чекпойнти (Checkpoints), які дають відчуття завершеності після кожної логічної частини.
  • 🏷️ Строга типізація: кожне завдання має унікальний ID (T001), маркер паралельності [P] та прив’язку до User Story ([US1]).
  • 🧹 Візуальна чистота: використання емодзі як візуальних якорів та лаконічні описи файлових шляхів.

Врахування Ручної Праці

Однією з ключових переваг нашого процесу є гнучкість. Оскільки я (як користувач) вже виконав оптимізацію витрат токенів та ініціалізував проект Astro вручну за допомогою шаблону minimal в папці engine, завдання T001 (Ініціалізація Astro) було автоматично позначене як виконане ([x]). Це дозволяє ШІ не марнувати час на кроки, які вже закриті.

Пріоритети Реалізації

Список завдань розділено на логічні фази:

  1. Setup & Foundational: Налаштування symlink для зв’язку engine та vault/pages, створення строгої Zod-схеми для контенту (включаючи нові поля type та state).
  2. CI/CD (US2): Створення .github/workflows/deploy.yml для автоматичного деплою на GitHub Pages.
  3. Zero-JS Astro (US3): Реалізація базового шаблону та динамічного роутингу.
  4. Контент (US1 & US4): Створення тестового контенту з правильним статусом (published).

Наступним кроком буде запуск безпосередньої реалізації (/speckit.implement). Завдяки такому детальному розбиттю, агенту потрібно лише брати завдання одне за одним, не гублячи контекст!
