SDD Journey: Від локальних білдів до автоматизованої State Machine

Сьогодні ми перетнули важливу межу — VadPoint.com офіційно в ефірі, а наш SDD-процес (Specification-Driven Development) пройшов бойове хрещення реальними інфраструктурними викликами.

🚀 Де ми зараз?

За останні кілька днів ми перетворили “просто статичний сайт” на автоматизовану видавничу платформу. Ось що було зроблено згідно з нашими специфікаціями:

1. Оновлення ядра до Node.js v24 (specs/003)

Astro v6 виявився вибагливим до оточення. Ми успішно оновили весь пайплайн до Node 24, що забезпечило стабільність білдів та доступ до найновіших можливостей двигуна.

2. Крос-репозиторний деплой (specs/005)

Це був один із найскладніших кроків. Ми розділили архітектуру на три частини:

  • vadpoint.com.dev: Наша приватна “кухня”, де зберігається код та Obsidian Vault.
  • GitHub Actions: Наш “шеф-кухар”, який готує сайт.
  • vadpoint.com: Публічна вітрина, де лежать тільки готові HTML-файли.

Завдяки використанню DEPLOY_PAT та параметру external_repository, ми налаштували автоматичну доставку контенту з приватного репозиторію у публічний без ручного втручання.

3. State Machine: Магія автоматизації

Ми реалізували справжній зворотний зв’язок між сервером та Obsidian:

  1. Ви ставите статус state: ready_for_publish у Markdown-файлі.
  2. Робите git push.
  3. GitHub Actions збирає сайт і, якщо все успішно, сам робить комміт назад, змінюючи статус на published.

Це дозволяє вам бачити реальний стан публікації прямо всередині Obsidian, не заходячи на GitHub.

🧠 Чому це важливо?

Ми дотримуємося принципу KISS (Keep It Simple, Stupid), але під капотом у нас працює складна логіка, яка робить життя автора простішим.

  • Markdown — єдине джерело істини.
  • Нуль ручної роботи після пуша коду.
  • Ізоляція: Публічний репозиторій — це просто “тупий” хостинг файлів, вся логіка надійно схована.

🛠 Що далі?

Тепер, коли фундамент та пайплайн залізобетонні, ми переходимо до естетики:

  • UI/UX Refinement: Створення преміального дизайну згідно з нашою Конституцією.
  • Lighthouse Excellence: Оптимізація до 100/100 балів.
  • Динамічні активи: Автоматична обробка зображень.

Ми продовжуємо наш шлях за SDD-протоколом. Кожна зміна — це специфікація, план, завдання і тільки потім код.

Stay tuned. The journey continues.
