Configuring PI Agent with Nvidia Nim

May 6, 2026 


### Важные нюансы для NIM:

1.  **Переменная в Shell:** Не забудьте экспортировать переменную в вашем `.zshrc` или `.bashrc`:
    ```bash
    export NVIDIA_NIM_API_KEY=nvapi-your-key-here
    ```
2.  **Специализированное расширение:** Существует официальное расширение для Pi — `pi-nvidia-nim`. Оно предпочтительнее ручной правки `models.json`, так как NIM использует нестандартные параметры для активации "мышления" (параметр `chat_template_kwargs` вместо `reasoning_effort`). 
    *   Установка: `pi install git:github.com/xRyul/pi-nvidia-nim`
    *   После этого NIM-модели подтянутся автоматически, а вам нужно будет только задать `NVIDIA_NIM_API_KEY`.

3.  **Контекст:** Хотя многие модели в NIM поддерживают 128k, в ручном конфиге лучше явно указывать `contextWindow`, чтобы Pi правильно планировал [Attention Budget](#) и не перегружал контекст сверх лимита бесплатного превью.

Для ваших задач с **Research Assistant (Feynman)** использование NIM через `models.json` — отличный способ получить доступ к топовым моделям (вроде Llama 3.1 405B или DeepSeek R1) сВ **Pi Coding Agent** (фреймворк `pi-mono`) работа с секретами реализована максимально гибко. Вы можете указывать имя переменной окружения напрямую в значении поля `apiKey`.

### 1. Как получить API KEY из переменной окружения

В файле `models.json` (обычно находится в `~/.pi/agent/models.json`) поле `apiKey` поддерживает автоматическое разрешение (resolution). Если значение совпадает с именем существующей переменной окружения, Pi подставит её значение.

**Варианты синтаксиса:**
*   **Обычная переменная:** `"apiKey": "MY_CUSTOM_KEY_ENV"` — Pi проверит наличие `process.env.MY_CUSTOM_KEY_ENV`.
*   **Shell-команда:** `"apiKey": "pass nvidia/nim-key"` — если вы используете менеджеры паролей вроде `pass`, Pi выполнит команду и подставит результат (удобно для безопасности).

---

### 2. Пример models.json для Nvidia NIM API

Nvidia NIM предоставляет OpenAI-совместимый интерфейс. Ниже приведен пример конфигурации, который регистрирует провайдера и конкретную модель (например, Llama 3.1 405B), используя переменную окружения `NVIDIA_NIM_API_KEY`.
```json
{
  "providers": {
    "nvidia": {
      "api": "openai",
      "baseUrl": "[https://integrate.api.nvidia.com/v1](https://integrate.api.nvidia.com/v1)",
      "apiKey": "NVIDIA_NIM_API_KEY",
      "compat": {
        "reasoning": true
      }
    }
  },
  "models": [
    {
      "id": "meta/llama-3.1-405b-instruct",
      "provider": "nvidia",
      "name": "NIM: Llama 3.1 405B",
      "contextWindow": 128000,
      "maxTokens": 4096
    },
    {
      "id": "deepseek-ai/deepseek-v3",
      "provider": "nvidia",
      "name": "NIM: DeepSeek V3",
      "contextWindow": 64000
    }
  ]
}

Важные нюансы для NIM:

  1. Переменная в Shell: Не забудьте экспортировать переменную в вашем .zshrc или .bashrc:

    export NVIDIA_NIM_API_KEY=nvapi-your-key-here

  2. Специализированное расширение: Существует официальное расширение для Pi — pi-nvidia-nim. Оно предпочтительнее ручной правки models.json, так как NIM использует нестандартные параметры для активации “мышления” (параметр chat_template_kwargs вместо reasoning_effort).

    • Установка: pi install git:github.com/xRyul/pi-nvidia-nim
    • После этого NIM-модели подтянутся автоматически, а вам нужно будет только задать NVIDIA_NIM_API_KEY.

  3. Контекст: Хотя многие модели в NIM поддерживают 128k, в ручном конфиге лучше явно указывать contextWindow, чтобы Pi правильно планировал Attention Budget и не перегружал контекст сверх лимита бесплатного превью.

Для ваших задач с Research Assistant (Feynman) использование NIM через models.json — отличный способ получить доступ к топовым моделям (вроде Llama 3.1 405B или DeepSeek R1) с минимальной задержкой.
